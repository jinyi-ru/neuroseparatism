## Установка

1. Поместите папку `neuroseparatism` в папку `custom_nodes` вашего **ComfyUI**
2. Перезапустите **ComfyUI**
3. В меню нод (nodes) появится категория **neuroseparatism**

## Доступные узлы

### 🔧 Qwen Image Edit Advanced
Основной узел для кодирования изображений с текстовыми инструкциями для Qwen-VL моделей.

**Входы:**
- `clip`: CLIP модель (обязательно)
- `prompt`: Текстовый промпт с инструкцией (обязательно)
- `image1`: Первое изображение (обязательно)
- `vae`: VAE модель (опционально, для генерации reference latents)
- `image2`, `image3`, `image4`: Дополнительные изображения (опционально)
- `multi_params`: Параметры обработки для каждого изображения (опционально)
- Глобальные параметры (используются если multi_params не заданы):
  - `global_vl_size`: Размер для vision encoder (по умолчанию 384)
  - `global_latent_size`: Размер для VAE encoder (по умолчанию 1024)
  - `global_crop_method`: Метод кадрирования (crop/pad/square_pad)
  - `global_resize_method`: Метод ресайза (lanczos/bilinear/bicubic/etc)
  - `global_raw_mode`: Raw режим для VAE (только кратность без ресайза)
  - `system_prompt`: Системный промпт для Qwen

**Выходы:**
- `conditioning`: Кондиционирование для Sampler
- `orig_width`: Исходная ширина первого изображения
- `orig_height`: Исходная высота первого изображения
- `info`: Информация о процессе обработки

### ⚙️ Qwen Processing Parameters
Узел для создания параметров обработки изображений.

**Входы:**
- `vl_target_size`: Целевой размер для vision encoder (8-2048, шаг 8)
- `latent_target_size`: Целевой размер для VAE (32-4096, шаг 32)
- `crop_method`: Метод приведения к кратности (crop/pad/square_pad)
- `raw_mode`: Raw режим (только для VAE)
- `resize_method`: Метод ресайза
- `params_name`: Имя параметров (опционально)

**Выходы:**
- `params`: Параметры обработки
- `info`: Информация о параметрах

### ⚙️ Qwen Multi-Image Parameters
Узел для объединения параметров обработки для нескольких изображений.

**Входы:**
- `image1_params`: Параметры для первого изображения (обязательно)
- `image2_params`, `image3_params`, `image4_params`: Параметры для дополнительных изображений (опционально)

**Выходы:**
- `multi_params`: Объединенные параметры для всех изображений
- `info`: Информация о наборе параметров

## Особенности

### Индивидуальные параметры для каждого изображения
Вы можете задать разные параметры обработки для каждого изображения:
1. Создайте несколько узлов **Qwen Processing Parameters** с разными настройками
2. Объедините их через **Qwen Multi-Image Parameters**
3. Подключите результат к основному узлу

### Два режима обработки изображений
1. **VL обработка**: Всегда масштабируется до заданного размера для vision encoder
2. **VAE обработка**: Может работать в raw_mode (только кратность) или с полным ресайзом

### Поддержка до 4 изображений одновременно
Узел поддерживает обработку до 4 изображений в одном промпте.

### Reference latents
При подключении VAE модели генерируются reference latents, которые можно использовать в расширенных сценариях генерации.

## Пример использования

1. Загрузите изображение через `Load Image`
2. Создайте параметры обработки через `Qwen Processing Parameters`
3. Подключите CLIP модель (поддерживающую Qwen)
4. Введите текстовый промпт с инструкцией
5. Подключите выход к Sampler

## Системный промпт по умолчанию

> Describe the key features of the input image (color, shape, size, texture, objects, background), then explain how the user's text instruction should alter or modify the image. Generate a new image that meets the user's requirements while maintaining consistency with the original input where appropriate.


Вы можете изменить системный промпт для изменения поведения модели.

## Технические детали

- Vision encoder всегда обрабатывается с кратностью 8
- VAE encoder обрабатывается с кратностью 32
- В raw_mode VAE только приводит размеры к кратности 32 без изменения масштаба
- Поддерживаются методы ресайза: nearest-exact, bilinear, area, bicubic, lanczos

### 3. ⚙️ Qwen Multi-Image Parameters (Настройки для нескольких фото)

**Что делает:** Объединяет разные настройки для каждого изображения.

**Пример:** У вас есть 3 фото. Для первого нужны одни настройки, для второго - другие. Этот узел позволяет это сделать.

## ⚡ Что означают настройки? (Для "чайников")

### 🎯 VL Target Size (Размер для Vision Encoder)
- **Что это:** Размер, до которого уменьшается изображение для Qwen
- **Как выбрать:** 
  - `256` - быстро, но мало деталей
  - `384` - баланс скорости и качества (по умолчанию)
  - `512` - больше деталей, но требует больше памяти
- **Важно:** Всегда кратность 8!

### 🔮 Latent Target Size (Размер для VAE)
- **Что это:** Размер, до которого уменьшается изображение для VAE
- **Как выбрать:**
  - `512` - быстро, подходит для простых правок
  - `1024` - стандарт (по умолчанию)
  - `1536` - больше деталей для сложных правок
- **Важно:** Всегда кратность 32!

### ✂️ Crop Method (Метод обрезки)
- **`crop`** - Обрезает края, чтобы вписать в нужный размер (лучше всего сохраняет композицию)
- **`pad`** - Добавляет чёрные поля по краям (сохраняет всё изображение)
- **`square_pad`** - Делает квадрат, добавляя поля (сохраняет всё изображение, медленнее)

### 🔥 RAW Mode (RAW-режим)
- **❌ Выключен:** Изображение масштабируется до нужного размера (быстрее)
- **✅ Включен:** Изображение только обрезается до кратности 32, без масштабирования (максимальное качество, медленнее)

### 🖼️ Resize Method (Алгоритм масштабирования)
1. **`lanczos`** - Лучшее качество, но медленно (по умолчанию)
2. **`bicubic`** - Хороший баланс
3. **`bilinear`** - Быстро, но может быть размыто
4. **`area`** - Для уменьшения размеров
5. **`nearest-exact`** - Пиксельный стиль (для pixel art)

## 🤖 Системный промпт: как и зачем его менять?
Системный промпт в Qwen задаёт **как модель должна думать и отвечать**, а не что именно делать с изображением. Это "инструкция по эксплуатации" для ИИ.

**Что значит системный промпт по умолчанию на русском:**
1. Сначала опиши ключевые особенности изображения
2. Потом объясни, как инструкция пользователя должна изменить изображение  
3. Сгенери новое изображение, которое соответствует требованиям
4. Сохраняй согласованность с оригиналом, где это уместно

---

## 🔧 Как и зачем менять системный промпт?

### Зачем менять?
- **Изменить стиль ответа** - сделать модель более креативной или технической
- **Сфокусировать на разных аспектах** - на эмоциях, композиции, деталях
- **Задать правила обработки** - что игнорировать, что подчеркнуть
- **Адаптировать под конкретные задачи** - для разных типов изображений

### Когда НЕ менять?
- Если устраивает стандартное поведение
- При работе с новыми типами запросов (сначала тестируйте стандартный)
- Если не уверены в эффекте

### 📋 Примеры готовых промптов (копируй и используй)
<details>

#### Пример 1: Для эмоционального анализа и преобразования
> First, analyze the emotional tone, mood, and atmosphere of the input image. Describe how colors, lighting, and composition contribute to the feeling. 
> Then, explain how the user's instruction will transform these emotional elements. Focus on conveying feelings and moods through visual changes. 
> Generate an image that not only follows the instructions but also powerfully communicates the intended emotional impact.


##### Что изменится:
- Модель будет больше внимания уделять эмоциям и настроению
- Инструкции будут про чувства, а не только про объект
- Результат будет более "атмосферным"

#### Пример 2: Для технического анализа и точных изменений
> Analyze the input image with technical precision: identify resolution, aspect ratio, color profiles, lighting conditions, and compositional rules used. 
> Provide a structured technical breakdown. Then, explain how the user's instruction will affect these technical parameters. 
> Focus on measurable changes and precise modifications. Generate an image that implements the instruction with technical accuracy.

##### Что изменится:
- Модель будет уделять больше внимания о разрешению, соотношению сторон, цветовым профилям
- Инструкции будут более структурированными и техническими
- Изменения будут более точными и предсказуемыми

#### Пример 3: Для креативных и экспериментальных преобразований
> Look at the input image with an artist's eye. Identify unique visual elements, interesting compositions, and creative potential. 
> Interpret the user's instruction as a creative challenge, not just a technical task. Suggest unexpected but coherent transformations. 
> Don't be afraid to break conventions if it serves the creative vision. Generate an image that surprises and inspires.

##### Что изменится:
- Модель будет предлагать более смелые и неожиданные решения
- Будет искать "творческий потенциал" в своих решениях
- Может нарушать правила, если это улучшит результат
</details>

### 💡 Как создавать свои системные промпты
- Структура хорошего системного промпта: `[Роль / подход]` + `[Что анализировать в первую очередь]` + `[Как обрабатывать инструкцию]` + `[Критерии результата]`

#### Пример разбора стандартного промпта:
- **Describe the key features**       ← ЧТО анализировать
- **then explain how**                ← КАК обрабатывать инструкцию  
- **Generate a new image that meets** ← КРИТЕРИИ результата
- **maintaining consistency**         ← ОГРАНИЧЕНИЯ/правила

## ⚠️ Важные замечания
- Длина имеет значение - слишком длинный промпт может "потеряться"
- Язык промпта - можно использовать и русский, но английский часто работает стабильнее
- Контекст - некоторые промпты лучше работают с определёнными типами изображений
- Тестирование - всегда проверяйте результат с новым промптом
Помните: Системный промпт - это мощный инструмент, который позволяет "программировать" поведение модели под ваши задачи!
